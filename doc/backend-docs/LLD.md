## 1. Models

**Socket initialization** (`src/socket/socket.ts`):
- Creates `GameManager` singleton
- Attaches all 10 handlers per connection event
- Starts 1-second interval broadcast for `turn-timer`
- Manages `turnTimeouts` map for per-room setTimeout tracking
- Exports `scheduleTurnTimeout`, `clearTurnTimeout`, `getIO`, `getGameManager`

## 3. Validation Logic

### 3.1 RoomManager Validations (`src/game/RoomManager.ts`)

**createRoom** — validates before room creation:
1. `validatePlayerNotInAnyRoom(playerId)` — throws `PlayerAlreadyInRoomError` if `playerToRoom` map already contains `playerId`
2. `validateSocketNotInAnyRoom(socketId)` — throws `DuplicateSocketError` if `socketToRoom` map already contains `socketId`

**joinRoom** — validates before joining:
1. `getRoomOrThrow(roomId)` — throws `RoomNotFoundError` if roomId not in `rooms` map
2. `validatePlayerNotInAnyRoom(playerId)` — same as above
3. `validateSocketNotInAnyRoom(socketId)` — same as above
4. `validateRoomNotFull(room)` — throws `RoomFullError` if `room.players.length >= 2`
5. `validatePlayerNotDuplicateInRoom(room, playerId)` — throws `DuplicatePlayerError` if playerId already exists in room.players
6. `validateSocketNotDuplicateInRoom(room, socketId)` — throws `DuplicateSocketError` if socketId already exists in room.players

**leaveRoom** — no throw, silently returns if:
- Player not found in any room (`getRoomByPlayerOrThrow` throws, but `leaveRoom` on `GameManager` catches)
- Player index -1 in room (internal guard)

**updatePlayerSocket** — validates:
- `getRoomByPlayerOrThrow(playerId)` — throws `PlayerNotInRoomError` if player not indexed
- Player exists in room.players — throws `PlayerNotInRoomError` if not found

**resetGame** — validates:
- `getRoomOrThrow(roomId)` — throws `RoomNotFoundError`

### 3.2 GameEngine Validations (`src/game/GameEngine.ts`)

**makeMove** — executes in order:

| Step | Condition | Error | Handler |
|---|---|---|---|
| 1 | `startedAt` is null (game not initialized) | `GameNotStartedError` | thrown to caller |
| 2 | `isDraw === true` or `winner !== null` | `GameAlreadyOverError` | thrown to caller |
| 3 | `row` not in [0,2] or `col` not in [0,2] | `InvalidMoveError("Position out of bounds")` | thrown to caller |
| 4 | `board[row][col] !== null` | `CellOccupiedError(row, col)` | thrown to caller |
| 5 | `currentPlayer` is null (game ended during request) | `GameNotStartedError` | thrown to caller |
| 6 | `currentPlayer.playerId !== playerId` | `NotYourTurnError` | thrown to caller |
| 7 | `turnDeadline` exists AND `Date.now() > turnDeadline` | If timeout enabled → `resolveTimeout()` (returns win for opponent). If disabled → `MoveTimeoutError` | thrown or returned |

Post-validation side effects:
- Symbol placed on board
- Move recorded in history
- `WinnerChecker.check(board)` evaluates:
  - 3 rows, 3 columns, 2 diagonals for 3-in-a-row match
  - If no winner and all cells filled → draw
- If win → `winner` set, `winningCells` recorded, `completedAt` set
- If draw → `isDraw` set, `completedAt` set
- If continue → flip `currentPlayerIndex` (0→1, 1→0), reset `turnDeadline`

### 3.3 GameManager Validations (`src/game/GameManager.ts`)

**makeMove** — pre-engine validation:
- `getRoomOrThrow(roomId)` — throws `RoomNotFoundError`
- `room.status !== RoomStatus.ACTIVE` → returns `MoveResult { success: false, error: "Game is not active" }`

Post-engine state:
- If result has winner or isDraw → `room.status = RoomStatus.FINISHED`

**playAgain** — validates:
- `getRoomOrThrow(roomId)` — throws `RoomNotFoundError`
- `room.status !== RoomStatus.FINISHED` → throws `Error("Game is not finished")`
- Player not in room.players → throws `Error("Player not in this room")`

**checkTimeout** — validates:
- Room not found or not ACTIVE → returns `null` (no-op)

### 3.4 Socket Handler Validations

**createRoom.handler** — catches `RoomError` from `gameManager.createRoom()` and emits `error` event

**joinRoom.handler** — catches `RoomError` from `gameManager.joinRoom()` and emits `error` event

**makeMove.handler**:
- Catches exceptions from `gameManager.makeMove()` (GameError or RoomError) and emits `move-result { success: false, error }` (NOT the `error` event)
- Valid move with win/draw → `clearTurnTimeout(roomId)`
- Valid move with continue → `scheduleTurnTimeout(roomId, delay)` if timeout enabled

**reconnect.handler**:
- `gameManager.reconnectPlayer()` returns null → emits `error { message: "No active game found" }`
- Caught exceptions → emit `error` event

**leaveRoom.handler**:
- `gameManager.leaveRoom()` returns null → emits `error { message: "You are not in a room" }`

**disconnect.handler**:
- If room FINISHED → `gameManager.leaveRoom()` (removes player entirely), emits `player-left`, clears timeout
- If room ACTIVE/OPEN → `gameManager.disconnectPlayer()` (marks disconnected), emits `player-disconnected`

**playAgain.handler** — catches errors, emits `error` event

### 3.5 Timer Validation

**Timer broadcast** (`startTimerBroadcast` in `socket.ts`):
- Skips rooms where `turnDeadline` is null, `currentPlayer` is null, or `game.isGameOver()` is true
- Computes `remainingMs = Math.max(0, turnDeadline - Date.now())`

**Timeout resolution** (`scheduleTurnTimeout` in `socket.ts`):
- `checkTimeout()` on the game engine validates that the game is started, not over, timeout enabled, and deadline has passed
- Only emits `move-result` with `timeoutWin: true` if timeout actually resolves

## 5. Key Internal Dependencies

```
socket.ts (handlers + timer)
    └── GameManager (facade)
            ├── RoomManager (storage + validation)
            └── GameEngine (game logic + validation)
                    └── WinnerChecker (stateless board eval)
```

- `GameManager` is the single public API consumed by all 10 socket handlers
- `scheduleTurnTimeout` / `clearTurnTimeout` in `socket.ts` handle timer lifecycle per room
- `startTimerBroadcast` runs as a global interval independent of per-room timeouts
- `RoomManager` uses 3 in-memory index maps: `rooms`, `playerToRoom`, `socketToRoom`
- All `RoomError` / `GameError` exceptions are caught in socket handlers and returned as typed events (either `error` or `move-result`)

## 6. Edge Case Handling Summary

| Scenario | Validation Layer | Action |
|---|---|---|
| Create room while already in room | RoomManager | `PlayerAlreadyInRoomError` → `error` event |
| Join non-existent room | RoomManager | `RoomNotFoundError` → `error` event |
| Join full room | RoomManager | `RoomFullError` → `error` event |
| Move on finished game | GameEngine → GameManager | `GameAlreadyOverError` → `move-result { success: false }` |
| Move out-of-bounds | GameEngine | `InvalidMoveError` → `move-result { success: false }` |
| Move on occupied cell | GameEngine | `CellOccupiedError` → `move-result { success: false }` |
| Move from wrong player | GameEngine | `NotYourTurnError` → `move-result { success: false }` |
| Move on non-ACTIVE room | GameManager | Return `success: false, error: "Game is not active"` |
| Expired turn (timeout enabled) | GameEngine | Auto-win for opponent via `resolveTimeout()` |
| Expired turn (timeout disabled) | GameEngine | `MoveTimeoutError` → `move-result { success: false }` |
| Play again on non-finished room | GameManager | `Error("Game is not finished")` → `error` event |
| Disconnect in FINISHED room | disconnect handler | Auto `leaveRoom` + `player-left` |
| Disconnect in ACTIVE room | disconnect handler | `player-disconnected` (player remains) |
| Reconnect after server restart | reconnect handler | `error { message: "No active game found" }` |
| Leave room with both players gone | RoomManager | Room deleted from map + all indexes cleared |
| Leave room in ACTIVE/FINISHED state | RoomManager | Room reverted to OPEN, game engine reset |
